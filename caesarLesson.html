So, you want to learn how to use the Caesar cipher to encrypt and decrypt messages? Well you have come to the right place!

The Caesar cipher operates using an integer key value which we will call the "shifting" value. We then use this value to change the character values in the message. We take each letter in the message and find its character value.

A character value of a letter corresponds to its position in the alphabet. In this lesson we will be working with the English alphabet (a-z). So the values of our letter would be as followed;

The letter "a" would have a character value of 1, "b" would have the value of 2, "c" would have the value of 3 and so on until we get to "z" which has the value of 26.

Now, back to our cipher and how we use it to encrypt a message. So we take the shifting value and add it to the character value of the letter in the message. This process changes the letters character code and therefor changes the letter to that of the letter with the new character value.

For example, we have the letter "m"and a shifting value of 4, we take the character value of "m", which is 13, and add 4 to it. This gives us a value of 17, the 17th letter in the alphabet is "q", so in our message we would replace every instance of "m" with "q".

We carry out the same process for every letter in the message or "plaintext", to encrypt completely and obtain the secret message or "ciphertext" which cannot be interpreted without the key/ shifting value.

To decrypt the ciphertext we simply take the key and subtract it from every letters character value. So, say we had the letter "o" and a key of 10, we would replace "o" with the letter that is 10 places earlier in the alphabet, which is "e".

Again, we carry out the same process for every letter in the ciphertext and will end up with the original message to read!

But what happens when the process of adding or subtracting the key from the character value gives a result that is out of range? I.e. the value is less than 1 or more than 26? Carry on to the next section to find out!



This section brings us to the key concept that allows this form of encryption to become possible, modular arithmetic.

Modular arithmetic, also known as clock face arithmetic, is a system of arithmetic where the numbers "wrap around" when it reaches a certain value. Basically, if the number is bigger than a specific max value then we will divide the bigger number by the max value and take the remainder as the new value.

Let us do an example of how this would work in the case of the Caesar cipher. The maximum value a letter could have in the English alphabet is 26. So, when performing encryption using the Caesar cipher, we would do the same as before, add the shifting value to the character value, but we add an extra step. After performing this addition we would then divide the new character value by the maximum value, 26, and would take the remainder as the new character value.

So, say we have the letter "x" and a shifting value of 11, the character value of "x" is 24, adding 11 to this would give us 35, which it too large so it does not correspond to a letter. To get the correct letter we do modular arithmetic. We add the character value and the key together, so 24+11 to give us 35. Next, we take that value of 34 and divide it by the max value (26).

34/26 = 1 with a remainder of 8

So, we take the remainder value of 8 and this is our new letter, the 8th letter of the alphabet, which is "h". We would carry out this step for any values that go over 26 after the addition of the key takes place.
