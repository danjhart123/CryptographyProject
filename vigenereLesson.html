Now it is time for lesson two on this site which is going to cover the topic of the vigenère cipher. We will cover what it is and how it works.

A vigenère cipher is basically a more advanced Caesar cipher because it uses more than one alphabet to swap out the letters, making it a "poly-alphabetic", substitution cipher. In fact, the vigenère cipher actually works in almost the same way as the Caesar cipher.

So how does it work? Like the Caesar cipher (and all other cryptographic ciphers) we need a key to allow us to change the values in the "plaintext". However, unlike the Caesar cipher, the key for the viginère needs to be a word/phrase, not a numerical value.

So the Vigenère cipher works using a string key and the Vigenère table which is pictured below.





As you can see, the Vigenère table is a 2-dimensional table where the rows and columns are both indexed by letters in the alphabet, and the contents of the table are 26 different variations of the English alphabet that has been shifted with the Caesar cipher by various shifting values. This table is used to encrypt a string using a string key word. So, how does it work?

Say we have a string we want to encrypt (the plaintext) and another string that we will use to encrypt the plaintext (the encryption key) , we would match up the letters in the plaintext with the letters in the encryption key. Once this is done, we would look at the pairs of letters that have been produced by this step.

We then take the letter from the plaintext and select the row with that letter from the table. Next, we take the corresponding key letter and select the column with that letter from the table. The final step is to swap the letter from the plaintext to the letter at the point where the row and column meet, we then repeat this process until all of the letters have been substituted.

Basically, we replace the letter in the plaintext with the letter in the table that is at the index point [plaintext character, key character].

To decrypt a phrase that has been encrypted in this way we just do the reverse. We take the key word and match it up with the ciphertext that we want to decrypt, and like in the encryption process, we then replace each letter in the ciphertext with the letter in the table that is at index point [ciphertext character, key character].

Let us look at an actual example, we will have a plaintext of "cryptography web app" and an encryption key of "hello". In the first step of this encryption we take the key "hello" and match it up against the letters in "cryptography web app" as shown below.

*letter outside bracket = character from plaintext
*letter inside bracket = character from key


c(h) r(e) y(l) p(l) t(o) o(h) g(e) r(l) a(l) p(o) h(h) y(e) w(l) e(l) b(o) a(h) p(e) p(l)



Once this has been done, we will perform encryption based off the pairs of letters and the Vigenère table. Our first pair in this encryption will be (c,h) where "c" is the first letter of the plaintext and "h" is the first letter of the key. We look at the Vigenère table and select the row "c" and the column "h" and find the letter at the point where these 2 meet (index [c,h]). In this case the new letter is "j" so we would replace the "c" from "cryptography web app" with a "j" so it becomes "jomputer science".

Let us continue the encryption below.

c = index[c,h] = j

r = index[r,e] = v

y = index[y,l] = j

p = index[p,l] = a

t = index[t,o] = h

o = index[o,h] = v

g = index[g,e] = k

r = index[r,l] = c

a = index[a,l] = l

p = index[p,o] = d

h = index[h,h] = o

y = index[y,e] = c

w = index[w,l] = h

e = index[e,l] = p

b = index[b,o] = p

a = index[a,h] = h

p = index[p,e] = t

p = index[p,l] = a



After this encryption the word "cryptography web app" becomes the ciphertext "jvjahvkcldoc hpp hta" which is completely different. This form of encryption is more secure than the simple Caesar cipher because without the key there is no way of knowing which variations of the alphabet was used to encrypt the word.

if you are still unsure how it works then it may be easier to think about it in a simpler way. A simpler way to perform this encryption is to take (the character value of the character in the key − 1) as the shifting value and simply shift the character of the plaintext by the shifting value. This is done using the Caesar cipher. For example, if the key had the letter "a" we would have a shifting value of (1-1) = 0, meaning we would not change the value. If it was "b" then the shifting value would be (2-1) = 1. We would then apply this to the rest of the alphabet so we would know the shifting values to be applied in the encryption depending on the key.
