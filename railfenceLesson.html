Let us begin your third lesson on the world of cryptography by learning about the rail-fence cipher, also know as the "zig zag". This cipher is the first transposition cipher we have come across so far so maybe we should first cover what a transposition cipher does.

In general, we have 2 main type of ciphers, substitution and transposition. We have already seen substitution ciphers in the for of the Caesar and vigenère cipher. Both of these ciphers replace the letters in a word with completely different letters from a specific alphabet. Transposition ciphers are different due to the fact that they do not replace the letters in a word but simply re-order the letters in a certain way depending on the cipher used. Basically a transposition cipher "jumbles" up the order of the letters to create the ciphertext.

So that is how transposition ciphers work in general, but how exactly does the rail-fence cipher work? Firstly, as with all ciphers covered so far, we need a key to be able to encrypt a word in this way. The key for this cipher is a number which represents the amount of "rails" that will be used in this encryption.

The amount of rails is basically how many rows we have to work with for encryption. To encrypt in this way, we take each letter of the plaintext and write it out diagonally, from left to right, with a letter in each row. We go downwards diagonally until we reach the bottom rail, then we go up diagonally doing the same until we reach the top, then repeat the process. We carry out this process until we have no more letters left in the word to encrypt.

Once we have completed this process it is time to create the ciphertext we require. To do this we simply write out the letters in the first rail, followed by the letters in the second rail, and then do this for all of the rails until we have written out all of the letters. The new order of the letters is the ciphertext in this cipher.

To decrypt a message encrypted in this way we need to know how many rails were involved in the encryption. Once we know this, we divide the length of the ciphertext by the number of rails. We can then split the letters in the ciphertext into groups that match the number of rails. So if we had 3 rails we would divide the length of the ciphertext by 3 and then start from the end of the ciphertext and write them out in a reverse "zig-zag".

In this section we will work through an actual example of the rail-fence cipher to help cement your understanding of how it works. In this example we will use the plaintext "this is a secret message" and a key of 4 (4 rails will be used). Look at the image below to visualise the example.

As you can see from the above image we take the letters in the plaintext and write them downwards diagonally from left to right. We do this until we hit the bottom rail and then start writing the letters out upwards diagonally from left to right until we hit the top rail, where we go downwards again.

We carry on doing this until all the letters have been used, at which point we have our rails that we can use to create our ciphertext.

The rails from this example are as followed:

Rail 1: {t, a, t, g}

Rail 2: {h, s, s, e, m, a, e}

Rail 3: {i, i, e, r, e, s}

Rail 4: {s, c, s}

To create our ciphertext we just write out the letters in each rail in ascending order (1 - 4). Once we do this we get the ciphertext:

tatghssemaeiieresscs
