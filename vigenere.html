Vigenère Cipher

The Vigenère cipher is another substitution cipher like the Caesar cipher but this one uses multiple alphabets for encryption, so it is known as poly-alphabetic. This cipher is not just similar to the Caesar cipher in the fact that it substitutes the letters in a word for a different one, but the Vigenère cipher actually works by using 26 different variations of the Caesar cipher.

Now one of the key differences between the 2 ciphers is that the Vigenère cipher uses a string as a key for encryption whereas the Caesar cipher uses an integer "shifting" value as a key for encryption. So the Vigenère cipher works using a string key and the Vigenère table which is pictured below.





As you can see, the Vigenère table is a 2-dimensional table where the rows and columns are both indexed by letters in the alphabet, and the contents of the table are 26 different variations of the Caesar cipher, this table is used to encrypt a string using a string key word. So, how does it work?

Say we have a string we want to encrypt (the plaintext) and another string that we will use to encrypt the plaintext (the encryption key) , we would match up the letters in the plaintext with the letters in the encryption key. Once this is done, we would look at the pairs of letters that have been produced by this step.

We then take the letter from the plaintext and select the row with that letter from the table. Next, we take the corresponding key letter and select the column with that letter from the table. The final step is to swap the letter from the plaintext to the letter at the point where the row and column meet, we then repeat this process until all of the letters have been substituted.

Let us look at an actual example, we will have a plaintext of "computer science" and an encryption key of "hello". In the first step of this encryption we take the key "hello" and match it up against the letters in "computer science" as shown below.

*letter outside bracket = character from plaintext
*letter inside bracket = character from key



c(h) o(e) m(l) p(l) u(o) t(h) e(e) r(l) s(l) c(o) i(h) e(e) n(l) c(l) e(o)



Once this has been done, we will perform encryption based off the pairs of letters and the Vigenère table. Our first pair in this encryption will be (c,h) where "c" is the first letter of the plaintext and "h" is the first letter of the key. We look at the Vigenère table and select the row "c" and the column "h" and find the letter at the point where these 2 meet (index [c,h]). In this case the new letter is "j" so we would replace the "c" from "computer science" with a "j" so it becomes "jomputer science". Let us continue the encryption below.

c = index[c,h] = j

o = index[o,e] = s

m = index[m,l] = x

p = index[p,l] = a

u = index[u,o] = i

t = index[t,h] = a

e = index[e,e] = i

r = index[r,l] = c

s = index[s,l] = d

c = index[c,o] = q

i = index[i,h] = p

e = index[e,e] = i

n = index[n,l] = y

c = index[c,l] = n

e = index[e,o] = s



After this encryption the word "computer science" becomes the ciphertext "jsxaiaic dqpiyns" which is completely different. This form of encryption is more secure than the simple Caesar cipher because without the key there is no way of knowing which variations of the alphabet was used to encrypt the word.

To perform decryption of the message you need to know the key and know how to use the Vigenère table to look up the values. we do the same matching up process with the ciphertext and the key like we did in the encryption process and then find the plaintext by looking at the corresponding values at the index point [ciphertext character, key character].

A simpler way to perform this encryption is to take (the character value of the character in the key − 1) as the shifting value and simply shift the character of the plaintext by the shifting value. This is done using the Caesar cipher. For example, if the key had the letter "a" we would have a shifting value of (1-1) = 0, meaning we would not change the value. If it was "b" then the shifting value would be (2-1) = 1. We would then apply this to the rest of the alphabet so we would know the shifting values to be applied in the encryption depending on the key.
